This web server works similarly to When2Meet. Each databse entry has a unique id, as well as a first_name, start_time, end_time, week_day, and is_deleted. The idea is that a user will add to the database the times that they are busy and their friends are able to see these times, too.

Currently, the database (named 'schedule') contains: {"id":1,"first_name":"John","start_time":"14:00:00","end_time":"17:30:00","week_day":"Wednesday","is_deleted":0}
This web server has insert, update, patch and delete endpoints. To test my endpoints:

For INSERT, add to the schedule database like this:
curl --header 'Content-Type: application/json' \
--data '{"first_name": "Jane", "start_time": "8:00", "end_time": "10:45", "week_day": "Tuesday", "is_deleted": 0}' \
https://project2.le-sage.me:8443/schedule
The id is serial, so don't worry about specifiying one for the new entry (MySql will do it for you).

For SELECT, there are a few options:
You can curl https://project2.le-sage.me:8443/schedule to see all schedule entries
Or curl https://project2.le-sage.me:8443/schedule/John/Wednesday to see a specific user's schedule for a specific day
curl https://project2.le-sage.me:8443/schedule/user/John for a user's schedule
Similarly, curl https://project2.le-sage.me:8443/schedule/id/1 to search via an entry id
Then lastly, curl https://project2.le-sage.me:8443/schedule/day/Wednesday to see schedules for a chosen day

For UPDATE, here is an example of updating the start and end times for Jane:
curl --header 'Content-Type: application/json' \
--request PATCH \
--data '{"first_name": "Jane", "start_time": "8:15", "end_time": "11:00", "week_day": "Tuesday"}' \
https://project2.le-sage.me:8443/schedule/2
Note that the entry id is needed to update the schedule.

For DELETE, only the schedule entry id should be specified:
curl --request DELETE \
https://project2.le-sage.me:8443/schedule/1
